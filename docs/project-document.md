# โครงการ Smart CoE Monitoring

ระบบ IoT สำหรับตรวจวัดและรวบรวมข้อมูลสภาพแวดล้อมจากห้องต่างๆ ภายในอาคารภาควิชาวิศวกรรมคอมพิวเตอร์ (CoE) ข้อมูลทั้งหมดจะถูกส่งขึ้นระบบ Cloud เพื่อนำไปแสดงผล วิเคราะห์พยากรณ์ล่วงหน้า และคำนวณต้นทุนพลังงานแบบเรียลไทม์

---

## 1. อุปกรณ์ฮาร์ดแวร์และเซ็นเซอร์ (Hardware & Sensor Nodes)

แต่ละห้องจะถูกติดตั้ง **Multimodal Board** (ESP32) ทำหน้าที่เป็นโหนดสำหรับวัดค่าสภาพแวดล้อมต่างๆ โดยรองรับเซ็นเซอร์ดังต่อไปนี้:

| เซ็นเซอร์        | หน้าที่                | ฟิลด์ข้อมูล (SensorType)    |
| ---------------- | ---------------------- | --------------------------- |
| **DHT11/22**     | วัดอุณหภูมิและความชื้น | `temperature`, `humidity`   |
| **LDR Sensor**   | วัดความสว่างของแสง     | `light` (0 = ปิด, 1 = เปิด) |
| **Sound Sensor** | ตรวจจับระดับเสียง      | `sound` (หน่วย dB)          |

---

## 2. โครงสร้างเครือข่ายและเส้นทางการส่งข้อมูล (Network Topology)

ระบบถูกออกแบบให้มีการส่งข้อมูลแบบลำดับชั้น (Hierarchical) เพื่อความเสถียร:

### ชั้น 2: ESP-NOW & Local Gateway

- **จุดติดตั้ง**: ห้อง R200, R201, และ **AI Co-Working Space (Co_Ai)**
- **การส่งข้อมูล**: โหนดเซ็นเซอร์ส่งข้อมูลผ่านโปรโตคอล **ESP-NOW** มายัง Sensor Gateway (ตั้งอยู่ที่ห้อง Co_CoE) ซึ่งจะแปลงข้อมูลเป็น MQTT และส่งต่อไปยัง Cloud

### ชั้น 3: High Availability nRF Gateway

- **จุดติดตั้ง**: ห้อง **R303**
- **การส่งข้อมูล**: ใช้โปรโตคอล **nRF24L01** ส่งข้อมูลไปยัง Dual Gateway ที่ห้อง **PUPA** เพื่อทำระบบสำรอง (Redundancy) หากเกตเวย์ตัวใดตัวหนึ่งขัดข้อง ระบบยังคงทำงานได้ต่อเนื่อง

---

## 3. สถาปัตยกรรมระบบคลาวด์ (Cloud Architecture)

โครงการใช้สถาปัตยกรรม **AWS Serverless IoT** เต็มรูปแบบ:

1. **AWS IoT Core**: รับข้อมูล MQTT จาก Edge Gateways ผ่าน mTLS Authentication
2. **AWS IoT Rules Engine**: คัดกรองและส่งต่อข้อมูลไปยัง Lambda
3. **AWS Lambda**: ประมวลผลข้อมูล (Validation/Transformation) และจัดการ API Requests
4. **Amazon DynamoDB**: จัดเก็บข้อมูลรูปแบบ Time-series (Partition Key: `DeviceId`, Sort Key: `Timestamp`)
5. **AWS API Gateway**: ให้บริการ REST API สำหรับ Dashboard

---

## 4. ฟีเจอร์หลักของเว็บแอปพลิเคชัน (Web Application Features)

Web App พัฒนาด้วย **Next.js** พร้อมดีไซน์แบบ **Claymorphism** มีฟีเจอร์หลักดังนี้:

- **Dashboard**: แสดงค่าปัจจุบันแบบ Real-time พร้อมกราฟแนวโน้ม 24 ชั่วโมง
- **History Dashboard**: ค้นหาข้อมูลย้อนหลังตามช่วงเวลา ห้อง และประเภทเซ็นเซอร์ พร้อมฟังก์ชันส่งออกข้อมูลเป็น CSV
- **Energy & Cost Analytics**: วิเคราะห์การใช้พลังงานไฟฟ้า (kWh) และคำนวณค่าไฟประมาณการตามอัตรา MEA/PEA (ค่าไฟฐาน + ค่า Ft + VAT)
- **Room Images**: แสดงรูปภาพสภาพภายในห้องเรียนผ่านระบบกล้อง ESP-CAM

---

## 5. การติดตั้งและจุดใช้งานปัจจุบัน

ปัจจุบันระบบรองรับห้องใช้งานหลักดังนี้:

- **R200**: ห้องปฏิบัติการคอมพิวเตอร์
- **R201**: ห้องปฏิบัติการคอมพิวเตอร์
- **R303**: ห้องปฏิบัติการวิจัย
- **AI Co-Working Space (Co_Ai)**: พื้นที่ทำงานร่วมกันด้าน AI

_หมายเหตุ: ห้อง COE, AIE, NETWORK และ R302 ถูกนำออกจากระบบการแสดงผลปัจจุบันเพื่อเน้นพื้นที่ใช้งานหลัก_
