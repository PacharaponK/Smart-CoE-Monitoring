# รายละเอียดการคำนวณพลังงานและค่าไฟฟ้า (Energy & Cost Calculation Guide)

เอกสารฉบับนี้อธิบายอัลกอริทึมและที่มาของตัวเลขที่ใช้ในแท็บ **"พลังงานและค่าไฟ"** เพื่อให้ผู้ใช้เข้าใจถึงกระบวนการแปลงข้อมูลดิบจากเซ็นเซอร์เป็นค่าใช้จ่ายจริง

---

## 1. การแปลงกำลังไฟฟ้าเป็นพลังงาน (Power to Energy)

เนื่องจากเซ็นเซอร์ส่งค่ามาเป็น **กำลังไฟฟ้า (Watt)** ในช่วงเวลาใดเวลาหนึ่ง แต่การคิดค่าไฟต้องใช้ **หน่วยพลังงาน (kWh)** เราจึงใช้การคำนวณแบบสะสมพื้นที่ใต้กราฟด้วยกฎสี่เหลี่ยมคางหมู (Trapezoidal Rule)

### สูตรการคำนวณ:
$$kWh = \sum \left( \frac{P_{prev} + P_{curr}}{2} \times \frac{\Delta T}{3600} \right) / 1000$$

**คำอธิบายตัวแปร:**
- $P_{curr}$ : ค่ากำลังไฟฟ้าปัจจุบัน (Watt)
- $P_{prev}$ : ค่ากำลังไฟฟ้าก่อนหน้า (Watt)
- $\Delta T$ : ระยะเวลาห่างกันระหว่าง 2 ข้อมูล (วินาที)
- $3600$ : แปลงวินาทีเป็นชั่วโมง
- $1000$ : แปลง Watt เป็น Kilowatt

---

## 2. กฎการประมาณการกรณีไม่มีเซ็นเซอร์พลังงาน (Mapping Rule)

หากอุปกรณ์ใดไม่มีเซ็นเซอร์ `power` หรือ `energy` โดยตรง ระบบจะใช้ **สถานะไฟส่องสว่าง (Light Status)** มาคำนวณแทนตามเงื่อนไขดังนี้:
- **สถานะเป็น 1 (เปิดไฟ):** สมมติว่าใช้กำลังไฟ **100W** (ค่าเฉลี่ยหลอดไฟและอุปกรณ์ในห้องเรียน)
- **สถานะเป็น 0 (ปิดไฟ):** สมมติว่าใช้กำลังไฟ **0W**

---

## 3. โครงสร้างอัตราค่าไฟฟ้า (Thai Electricity Tariff)

ระบบอ้างอิงการคิดค่าไฟตามประเภท **1.1.2 บ้านอยู่อาศัย/อาคารขนาดเล็ก** (ใช้ไฟฟ้าเกิน 150 หน่วยต่อเดือน) ของ กฟน./กฟภ.

### 3.1 ค่าไฟฟ้าฐาน (Base Charge) แบบขั้นบันได
| ช่วงหน่วย (kWh) | อัตรา (บาท/หน่วย) |
| :--- | :--- |
| 1 - 150 หน่วยแรก | 3.2484 |
| 151 - 400 หน่วยถัดไป | 4.2233 |
| หน่วยที่ 401 เป็นต้นไป | 4.4217 |

### 3.2 ค่าไฟฟ้าผันแปร (Ft)
- **สูตร:** $TotalKWh \times FtRate$
- *ค่าเริ่มต้นในระบบ:* 0.3972 บาท/หน่วย (ปรับเปลี่ยนได้ที่หน้าจอ)

### 3.3 ค่าบริการรายเดือน (Service Charge)
- เป็นค่าคงที่ตามประเภทผู้ใช้ไฟ
- *ค่าเริ่มต้นในระบบ:* 38.22 บาท (ปรับเปลี่ยนได้ที่หน้าจอ)

### 3.4 ภาษีมูลค่าเพิ่ม (VAT)
- คิดอัตรา **7%** จากยอดรวมของ (ค่าไฟฐาน + Ft + ค่าบริการ)

---

## 4. สูตรการสรุปยอดบิล (Total Bill Formula)

$$SubTotal = BaseCharge + FtCharge + ServiceCharge$$
$$TotalBill = SubTotal + (SubTotal \times 7\%)$$

---

## 5. การจัดการความถูกต้องของข้อมูล (Data Integrity)

เพื่อให้การคำนวณแม่นยำและไม่เกิดค่าไฟพุ่งสูงผิดปกติ ระบบมีกลไกป้องกันดังนี้:
1. **Max Gap Protection:** หากข้อมูลดิบหายไปเกิน **1 ชั่วโมง** ระบบจะไม่คำนวณค่าไฟในช่วงรอยต่อนั้น (เพื่อป้องกันการคูณพื้นที่ใต้กราฟที่ข้อมูลขาดหาย)
2. **NaN Protection:** มีการดักจับค่าที่ไม่ใช่ตัวเลข (NaN) ในทุกขั้นตอน และจะเปลี่ยนเป็น 0 ทันทีเพื่อป้องกันระบบล่ม
3. **Timezone Handling:** รองรับการคำนวณเวลาแบบ Local Time ของประเทศไทยเพื่อให้สอดคล้องกับวันในปฏิทินจริง

---

## 6. กระบวนการจัดการข้อมูล (Data Handling Pipeline)

เพื่อให้ระบบทำงานได้รวดเร็วและทนทานต่อข้อมูลที่อาจจะไม่สมบูรณ์ ระบบมีขั้นตอนการจัดการข้อมูลดังนี้:

### 6.1 กลยุทธ์การดึงข้อมูล (Fetch Strategy)
- **Single-Load (GetAll):** ระบบดึงข้อมูลดิบจาก API ทั้งหมดมาเก็บไว้ในหน่วยความจำของบราวเซอร์ (State) ครั้งเดียว (สูงสุด 2000 รายการ) 
- **Local Processing:** เมื่อผู้ใช้เปลี่ยน "อุปกรณ์" หรือ "ช่วงเวลา" ระบบจะคำนวณใหม่จากข้อมูลที่มีอยู่แล้วทันทีโดยไม่จำเป็นต้องยิง API ใหม่ (ลดภาระเซิร์ฟเวอร์และเพิ่มความเร็ว)

### 6.2 การปรับมาตรฐานข้อมูล (Data Normalization)
เนื่องจากข้อมูลจากอุปกรณ์ต่างรุ่นอาจมีชื่อฟิลด์ต่างกัน ระบบจะปรับให้เป็นมาตรฐานเดียวกันก่อนคำนวณ:
- **Device ID:** รองรับทั้ง `DeviceId` และ `deviceId` (และทำการเปรียบเทียบแบบไม่สนตัวพิมพ์ใหญ่-เล็ก)
- **Timestamp:** แปลงค่า `Timestamp` หรือ `timestamp` ให้เป็นตัวเลข Unix Epoch เพื่อใช้เปรียบเทียบเวลา
- **Sensor Type:** แปลงเป็นตัวพิมพ์เล็กทั้งหมดเพื่อการกรองข้อมูลที่แม่นยำ (`power`, `energy`, `light`)

### 6.3 ลำดับการประมวลผล (Processing Steps)
1. **Filtering:** กรองข้อมูลเฉพาะอุปกรณ์ที่เลือกและอยู่ในช่วงเวลาที่กำหนด
2. **Pre-filling Dates:** สร้างถังข้อมูล (Buckets) สำหรับทุกวันในช่วงที่เลือกไว้ล่วงหน้า (แม้จะไม่มีข้อมูลในวันนั้นเลยก็ตาม) เพื่อให้กราฟแสดงผลต่อเนื่องไม่กระโดด
3. **Sorting:** เรียงลำดับข้อมูลตามเวลาจากอดีตไปปัจจุบัน (สำคัญมากสำหรับการหาค่า $\Delta T$)
4. **Validation:** ตรวจสอบค่า `SensorValue` หากพบว่าเป็นค่าว่างหรือ `NaN` จะถูกข้ามไปโดยไม่นำมาคำนวณ
5. **Integration:** นำคู่ข้อมูลที่อยู่ติดกันมาหาพื้นที่ใต้กราฟ (Trapezoidal) โดยเช็คเงื่อนไข `MAX_GAP` (1 ชม.)
6. **Final Mapping:** แปลงผลลัพธ์เป็นวัตถุ `summary` และ `dailySeries` ที่พร้อมส่งให้ Component UI แสดงผล

## 7. จากข้อมูลดิบสู่พลังงานต่อชั่วโมง (Discrete Data to Energy)

หัวข้อนี้ตอบคำถามที่ว่า **"เมื่อข้อมูลส่งมาเป็นจุดเวลา (Snapshots) ระบบรู้ได้อย่างไรว่า 1 ชั่วโมงทำงานไปเท่าไหร่?"**

### 7.1 อุปมาอุปไมย (The Analogy)
การคำนวณนี้เหมือนกับการที่คุณขับรถแล้วมองเข็มไมล์ (ความเร็ว/กำลังไฟ) ทุกๆ 1 นาที:
- แม้คุณจะไม่ได้มองเข็มไมล์ตลอดเวลา แต่ถ้าคุณรู้ความเร็ว ณ นาทีที่ 1 และนาทีที่ 2 คุณจะสามารถประมาณ "ระยะทาง" (พลังงาน) ที่วิ่งไปได้ในช่วง 1 นาทีนั้น

### 7.2 ขั้นตอนทางคณิตศาสตร์
ระบบใช้เทคนิคการคำนวณพื้นที่ใต้กราฟที่เรียกว่า **Trapezoidal Integration**:

1.  **จับคู่ข้อมูล (Pairing):** นำข้อมูลปัจจุบัน ($P_{curr}$) มาจับคู่กับข้อมูลก่อนหน้า ($P_{prev}$)
2.  **หาความห่างของเวลา ($\Delta T$):** นำ `Timestamp` มาลบกัน (หน่วยเป็นชั่วโมง)
3.  **หาค่าเฉลี่ยกำลังไฟ:** $\frac{P_{prev} + P_{curr}}{2}$ (เพื่อหาค่ากลางในช่วงเวลานั้น)
4.  **แปลงเป็นหน่วยพลังงาน:** $kWh_{slice} = \left( \frac{AveragePower}{1000} \right) \times \Delta T$
5.  **สะสมผลลัพธ์ (Accumulation):** นำค่าที่ได้จากทุกๆ คู่ข้อมูลมาบวกกันจนครบ 1 ชั่วโมง หรือ 1 วัน

### 7.3 การจัดการกรณีข้อมูลขาดหาย
- **ข้อมูลถี่:** เช่น ส่งทุก 10 วินาที $\rightarrow$ พื้นที่ใต้กราฟจะแม่นยำสูงมาก
- **ข้อมูลห่าง:** เช่น ส่งห่างกัน 30 นาที $\rightarrow$ ระบบยังคำนวณให้ได้โดยใช้ค่าเฉลี่ย
- **ข้อมูลขาดหาย (Gap):** หากห่างกันเกิน **1 ชั่วโมง** ระบบจะถือว่า "ไม่สามารถเชื่อถือข้อมูลในช่วงรอยต่อได้" และจะหยุดคำนวณพลังงานสะสมชั่วคราว จนกว่าจะมีข้อมูลใหม่เข้ามา เพื่อป้องกันไม่ให้ค่าไฟสูงเกินจริงจากการเหมาค่าเฉลี่ยในเวลาที่ยาวเกินไป

### 7.4 การรองรับข้อมูลความละเอียดสูง (High-Resolution Support)
หากเซ็นเซอร์ส่งข้อมูลมาถี่ (เช่น ทุกๆ 10 วินาที) ระบบจะประมวลผลดังนี้:
- **ประมวลผลทุกจุด:** ระบบไม่ข้ามข้อมูลใดๆ โดยจะวน Loop คำนวณ "ชิ้นส่วนพลังงาน" ของทุกๆ 10 วินาทีนั้น
- **ความแม่นยำสูง:** การคำนวณพื้นที่ใต้กราฟจะมีความละเอียดสูงมาก ทำให้เห็นภาพรวมการใช้ไฟที่สะท้อนความเป็นจริง แม้จะมีการเปิด-ปิดอุปกรณ์ไฟฟ้าในช่วงเวลาสั้นๆ ก็ตาม

## 8. ตัวอย่างโค้ดที่ใช้งานจริง (Real Code Logic)

เพื่อให้เห็นภาพการจัดการข้อมูลทุก 10 วินาที นี่คือส่วนหนึ่งของโค้ดในไฟล์ `energy-utils.js`:

### 8.1 การวนลูปประมวลผลทุกจุดข้อมูล
ระบบใช้ `for loop` เพื่อเข้าถึงข้อมูลทุกชุดที่ดึงมาจากฐานข้อมูล ไม่มีการข้ามจุด:

```javascript
// วนลูปดูข้อมูลทุกแถวที่เรียงลำดับเวลาไว้แล้ว
for (let i = 0; i < sorted.length; i++) {
  const item = sorted[i];
  const val = parseFloat(item.SensorValue); // ค่า Watt ณ จุดนี้
  
  if (type === "power" && i > 0) {
    const prev = sorted[i-1]; // อ้างอิงจุดก่อนหน้า
    const t1 = new Date(prev.Timestamp).getTime();
    const t2 = new Date(item.Timestamp).getTime();
    const deltaHours = (t2 - t1) / (1000 * 3600); // ช่วงเวลาห่างกัน
    
    // คำนวณพื้นที่สี่เหลี่ยมคางหมูระหว่าง 2 จุด
    const avgPowerW = (parseFloat(prev.SensorValue) + val) / 2;
    const kwhDelta = (avgPowerW / 1000) * deltaHours;
    
    energySum += kwhDelta; // บวกสะสมพลังงานทีละนิด
  }
}
```

### 8.2 การคำนวณค่าไฟฐานแบบขั้นบันได (Tiered Calculation)
ฟังก์ชันที่ใช้เลียนแบบการคิดเงินของการไฟฟ้า:

```javascript
export function calculateBaseCharge(totalKWh) {
  let charge = 0;
  let remaining = totalKWh;

  // 150 หน่วยแรก
  const tier1 = Math.min(remaining, 150);
  charge += tier1 * 3.2484;
  remaining -= tier1;

  // 250 หน่วยถัดไป
  if (remaining > 0) {
    const tier2 = Math.min(remaining, 250);
    charge += tier2 * 4.2233;
    remaining -= tier2;
  }
  
  // ส่วนที่เกินจาก 400 หน่วย
  if (remaining > 0) {
    charge += remaining * 4.4217;
  }
  return charge;
}
```
